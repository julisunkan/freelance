Build a full-stack web application using Python Flask and SQLite called:

"NDA Generator – PWA with E-Sign, Templates, and Groq AI Enhancement"

---

# 🧠 CORE GOAL

Create a production-grade NDA generator that:
- Generates NDAs using 20 built-in templates
- Enhances NDA text using Groq AI (LLM integration)
- Has a secure admin page to configure Groq API settings
- Supports e-signatures via token-based links (NO login system)
- Generates PDFs server-side only (NO frontend blob downloads)
- Works as a Progressive Web App (PWA)
- Uses inline notifications only (NO popups/modals)
- Tracks audit logs for all actions
- Uses secure token-based document access

---

# 🤖 GROQ AI INTEGRATION (IMPORTANT FEATURE)

## Admin Configuration Page

Create a protected route:

/julisunkan

This page allows configuration of:

- GROQ_API_KEY
- GROQ_MODEL (e.g. llama-3.1-70b-versatile)
- AI_ENHANCEMENT_ENABLED (true/false)
- DEFAULT_TONE (formal / strict / startup / legal-heavy)

Store config in SQLite table:

CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);

## AI USAGE (CORE LOGIC)

Use Groq API to improve NDA content:

### When generating NDA:
Send prompt to Groq:

- party details
- selected template
- clause toggles
- jurisdiction
- purpose

### AI returns:
- improved NDA wording
- legally stronger phrasing
- optional risk warnings (NOT legal advice, just suggestions)

Use requests:

Headers:

Model:
- configurable from admin panel

# ✍️ E-SIGN SYSTEM (NO LOGIN)

Generate for each NDA:
- public_id
- sign_token_a
- sign_token_b

Routes:
- /nda/view/<public_id>
- /nda/sign/a/<token>
- /nda/sign/b/<token>

Store:
- signature image (canvas base64)
- IP address
- timestamp

Lock NDA after both signatures.

---

# 📤 PDF GENERATION (SERVER-SIDE ONLY)

Route:
- /nda/download/<public_id>

Use WeasyPrint or ReportLab.

Must include:
- AI-enhanced NDA text
- signatures
- timestamps
- audit info

Return via:
Flask send_file(..., as_attachment=True)

NO frontend blob downloads allowed.
